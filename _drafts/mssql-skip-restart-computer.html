Oi, papo rápido aqui. Achei algumas anotações (bem) antigas e acho que valem o post.

Ao instalar o aplicar atualização do SQL Server, podemos ter a seguinte checagem de regra impedindo a continuação do procedimento:

Uma das formas de contornar isso (e é um método extremamente divulgado e antigo) é apagando o conteúdo do registro PendingFileRename:

Computador\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Session Manager

Como esse registro fica vazio, a checagem de regra passa e consegue prosseguir:

Esse é o modo "interface gráfica" de resolver a coisa. Outras possibilidades via linha de comando:

setup.exe /SkipRules=RebootRequiredCheck /Action=Install

setup.exe /SkipRules=RebootRequiredCheck /Action=Patch

Para quem não sabia desses métodos, está aí uma informação técnica relativamente útil e truque bem conhecido. Mas esse não é o intuito do post obviamente :)

Qual é o porém?

Existe um motivo pelo qual o Windows mantém valores nessa chave e o motivo é que após aplicação de alguma instalação, atualização ou até mesmo execução de rotinas internas, o proprio sistema operacional deixa agendado para renomear ou excluir algum arquivo. Ou seja, essa chave não é específica de alterações do SQL Server:

Portanto antes de apagar o conteúdo da chave, considere reiniciar o host para aplicar as operações pendentes se aplicável, ignorar as regras (comandos acima)
