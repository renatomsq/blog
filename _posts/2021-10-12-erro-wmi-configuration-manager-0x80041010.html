Problema

Erro ao abrir Configuration Manager (Classe inválida 0x80041010), a mensagem completa em texto é:

Não é possível conectar-se ao provedor WMI. Você não possui permissão ou o servidor não está acessível. Observe que você só pode gerenciar os servidores do SQL Server 2005 e posterior com o SQL Server Configuration Manager. Classe inválida [0x80041010]

Causa raíz

Quando você desinstala uma instância do SQL Server (meu caso!) , o provedor WMI correspondente à versão vai pra vala (é removido).

Solução

A solução é documentada pela própria Microsoft que é recompilar o provedor via linha de comando: https://docs.microsoft.com/pt-br/troubleshoot/sql/tools/error-message-when-you-open-configuration-manager

Em resumo, entre na pasta Shared de acordo com a versão do Configuration Manager na qual você está tendo problema (no meu caso é a 150, correspondente ao SQL Server 2019) via Prompt de Comando elevado e digite o comando abaixo:

mofcomp "sqlmgmproviderxpsp2up.mof"

Depois disso só abrir o Configuration Manager, funfa que é uma beleza. Da série problemas do dia a dia que só falhas de WMI proporcionam pra gente.
