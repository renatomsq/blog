Como todos sabem, o SQL Server 2016 trouxe diversas melhorias na engine e algumas delas extremamente bem vindas, por menores que possam parecer (tipo o comando TRUNCATE funcionando para partições).

Uma delas é sua compatibilidade mais abrangente com versões antigas, agora permitindo a compatibilidade mínima de três (3) versões inferiores à atual ao invés de duas (2) como nas últimas versões:

[caption id="attachment_2591" align="aligncenter" width="1024"] Matriz de compatibilidade do SQL Server com versões anteriores[/caption]

Em suma, o SQL Server 2016 suporta até o nível de compatibilidade do SQL Server 2008. Seguindo os padrões anteriores o mínimo de compatibilidade exigida seria o 110, que é o SQL Server 2012. Isso facilita cada vez mais as migrações de versões legadas e sem suporte, para uma versão atualizada e com suporte do fabricante.

Logo, para quem tem interesse em fazer a migração e ainda manter o nível de compatibilidade no 2008 por qualquer motivo que seja (mas supondo, Cardinality Estimator) pode achar interessante a notícia.
